## 🔄 自动化工作流说明

1.  **发布任务**: 在上方添加任务，标记为 `[ ] 待开发`，提交代码 (`git push`)。
2.  **Codex 开发**: GitHub Actions 自动运行，生成代码并提交 PR。
3.  **Claude 审查**: 本地运行 `local_pr_watcher.sh` 或手动检查，Claude 进行 Code Review。
4.  **反馈循环**:
    - ✅ **通过**: 合并 PR，将任务移至[已完成]。
    - ❌ **打回**: 在任务描述中添加 `**Review Feedback**: ...`，Codex 会自动读取并修复。

### 任务 1: 产品展示列表页面

**优先级**: 🟡 中

**状态**:

- [ ] 待开发
- [ ] 开发中
- [ ] 待审查
- [ ] 已完成

**需求描述**:
创建一个“最新产品”展示页面，以网格形式展示产品卡片。

**前端需求**:
- [ ] 页面路径: `/products`
- [ ] 创建 `ProductCard` 组件
    - 显示图片 (可使用占位图 `https://via.placeholder.com/300`)
    - 显示标题、价格 (红色高亮)、简短描述
    - "加入购物车" 按钮
- [ ] 布局: 响应式网格 (Grid)，桌面端 3 列，移动端 1 列
- [ ] 数据获取: 从后端 API 获取数据并渲染

**后端需求**:
- [ ] 接口: `GET /api/products`
- [ ] 数据: 返回一个包含 6-10 个模拟产品对象的数组
- [ ] 结构: `{ id, title, price, description, imageUrl }`

**验收标准**:
- [ ] 访问 /products 能看到产品列表
- [ ] 布局在不同屏幕下表现良好
- [ ] 接口返回 JSON 数据格式正确
